The 'background' on this page is a realisation of John Conway's Game of Life.

There are much better versions of this on the web; I only added it in to make the site have some actual content. Feel free to use the code (admittedly it's not great code) for yourself - you can reference me if you like, but not necessary.

There are 3 simple rules to this 'game'. If a cell is 'alive', it can die in 2 ways; overcrowding (having more than 3 neighbours), or solitude (having 1 or fewer neighbours). If a cell is considered 'dead', it can come to life if it has 3 neighbours.

Despite the simple rules, incredible complexity can arise - it has been shown that general computations can be performed on a Game of Life board.

My version below uses a random colour for the 'dead' cells, and the inverse colour for the alive. I initialise it with what Conway coined the 'glider'; this shape will continue to move forever, with no obstructions. You can also pause the game and trace over the cells using the mouse (may be tricky on mobile-type device), and restart to see the effect - usually some mess ending up in a stable state.

Disclaimer - there may occasionally be some odd behaviour, as I made this very quickly, and haven't bothered to debug it extensively.

Computer Projects

  • Exact solution counter for an Ising model with arbitrary couplings. That is, it generates the histogram of all energy levels for an Ising model. Actually, this code is probably not very useful as a whole (it was fairly specific to what I needed it for at the time, and unfortunately is not as general as it could be), but some parts of it may be helpful. The input files should be of form 'i j Jij' (tab separated). The function that does it all is called Count_Solutions. I never bothered to optimise this code (as I didn't need to), but I tried it on up to 31 bits, and it has no problem there.
